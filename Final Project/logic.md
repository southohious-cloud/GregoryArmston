# CS50P - Final Project Logic Pattern Summary Sheet

1. project.py - Main Program
Logic pattern: modular function orchestration
· Define main() in project.py
· Implement at least 3 custom functions
· Each function defined at top level (not nested)
· Use main() to orchestrate logic and I/O

2. test_project.py - Unit Tests
Logic pattern: pytest function matching
· Create test_project.py in root folder
· Write test_ functions for each custom function
· Use pytest to validate behavior
· Match test names to function names

3. requirements.txt - Dependencies
Logic pattern: pip install tracking
· List required libraries one per line
· Use pip freeze to verify
· Ensure file is in root folder

4. Design Expectations
Logic pattern: open-ended architecture
· Solve a real-world problem
· Use functions, classes, or modules as needed
· Validate inputs and handle errors
· Produce meaningful output or behavior
· Document usage and structure
